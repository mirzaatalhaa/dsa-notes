# Check Duplicate

## Problem
Given an array of integers, check if it contains any duplicate elements. Return `true` if any value appears at least twice, and `false` if every element is distinct.

## Approach
We use a **set**, since sets do not allow duplicate values.

We traverse the array once:
- For each number, check if it already exists in the set.
- If it exists → we found a duplicate → return `True` immediately.
- If it doesn't exist → add it to the set and continue.

If the loop completes without finding any duplicate, return `False`.

## Code
```python
class Solution:
    def hasDuplicate(self, nums: List[int]) -> bool:
        seen = set()
        for num in nums:
            if num in seen:
                return True
            else:
                seen.add(num)
        return False
```

## Complexity
- **Time:** O(n) — single pass through the array, set operations are O(1) on average.
- **Space:** O(n) — in the worst case (no duplicates), the set stores all elements.

## Mistakes Made & Fixes
1. **Indentation error** — the `if/else` block inside the loop wasn't properly aligned, causing logic errors.
2. **Wrong placement of `return False`** — initially placed inside the loop, which would return `False` after checking only the first element. It must be placed **after** the loop, so it only executes once all elements have been checked.

## Key Takeaways
- Sets are ideal for fast membership checks (O(1) average lookup).
- `return False` after a loop means "checked everything, found nothing" — placing it inside the loop breaks this logic.
- Always double-check indentation in Python since it directly affects control flow.
